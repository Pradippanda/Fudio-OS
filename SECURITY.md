Security Policy

Supported Versions

The following versions of Fudio OS are currently supported with security updates.

Version| Supported
Latest Release| ✅ Yes
Older Releases| ❌ No

---

Reporting a Security Vulnerability

The security of Fudio OS and its users is important.

If you discover a security vulnerability, please do not open a public GitHub Issue.

Instead, report it privately by contacting the project maintainer.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Screenshots or proof of concept (if applicable)
- Suggested fix (optional)

---

Response Process

We aim to:

- Acknowledge reports within 72 hours.
- Investigate the issue promptly.
- Keep reporters informed of progress.
- Release a fix as quickly as practical.
- Publicly disclose the vulnerability only after a fix is available.

---

Responsible Disclosure

Please avoid publicly sharing details of a security issue until it has been investigated and resolved.

We appreciate responsible disclosure and will acknowledge contributors who help improve the security of Fudio OS (unless they prefer to remain anonymous).

---

Security Best Practices

Contributors should:

- Never commit API keys, passwords, or secrets.
- Use environment variables for sensitive configuration.
- Keep dependencies up to date.
- Review third-party packages before adding them.
- Validate all user input where applicable.

Thank you for helping keep Fudio OS secure.
