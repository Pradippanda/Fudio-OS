Contributing to Fudio OS

First, thank you for your interest in contributing to Fudio OS. Every contribution—whether it's fixing a bug, improving documentation, or adding a new feature—helps make the project better.

---

Code of Conduct

By participating in this project, you agree to follow our Code of Conduct. Please be respectful, constructive, and welcoming to everyone.

---

Before You Start

Before working on a new feature or bug fix:

1. Check existing Issues to avoid duplicate work.
2. If your change is significant, open an Issue first to discuss the proposal.
3. Keep pull requests focused on a single improvement.

---

Development Setup

Clone the repository:

git clone https://github.com/Pradippanda/Fudio-OS.git
cd Fudio-OS

Install dependencies:

npm install

Run the development server:

npm run dev

---

Branch Naming

Please use descriptive branch names.

Examples:

feature/menu-management
feature/dashboard

fix/login-bug
fix/order-status

docs/readme-update
docs/api

refactor/sidebar

---

Commit Message Guidelines

Write clear and concise commit messages.

Examples:

feat: add inventory dashboard

fix: resolve login redirect issue

docs: improve installation guide

refactor: simplify menu component

---

Pull Request Checklist

Before submitting a Pull Request, make sure:

- [ ] Code builds successfully.
- [ ] Existing functionality is not broken.
- [ ] New code follows the project's style.
- [ ] Documentation is updated if needed.
- [ ] Screenshots are included for UI changes.
- [ ] Commit history is clean.

---

Reporting Bugs

When reporting a bug, include:

- Operating System
- Browser
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if available)

---

Suggesting Features

Feature requests should include:

- Problem statement
- Proposed solution
- Benefits
- Alternative approaches (if any)

---

Coding Standards

Please follow these principles:

- Write readable code.
- Keep functions small and focused.
- Use meaningful variable names.
- Remove unused code.
- Avoid unnecessary dependencies.
- Add comments only where they improve understanding.

---

Documentation

If your contribution changes functionality, update the relevant documentation and README where appropriate.

---

First-Time Contributors

If you're new to open source, look for Issues labeled:

- "good first issue"
- "help wanted"

These are designed to help new contributors get started.

---

Questions

If you have questions, open a GitHub Discussion or create an Issue before beginning large changes.

---

Recognition

All contributors are appreciated and will be recognized in the project's contributors list.

Thank you for helping improve Fudio OS.
